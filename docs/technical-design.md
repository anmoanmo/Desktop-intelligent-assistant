# 技术说明

## 项目定位

桌面智能体助手是一个桌面端大模型应用。它把桌面窗口、角色渲染、大模型对话、上下文感知、工具调用、权限确认和长期记忆组合在一起，形成一个可以本地运行的智能体原型。

项目重点不是单纯聊天，而是让模型能够在明确边界内理解用户当前桌面环境，并通过受控工具完成低风险辅助动作。

## 核心模块

- `settings.py`：加载 `.env`、TOML 配置和环境变量，形成类型化配置对象。
- `model_sources.py`、`models.py`、`model_registry.py`：解析模型来源目录，发现 Live2D 和 Spine 角色资源。
- `memory.py`、`profiles.py`：管理人设、长期记忆、对话记录和多角色档案。
- `desktop_context.py`：采集桌面上下文，macOS 下可读取前台应用和窗口信息。
- `tools.py`、`confirmations.py`、`audit.py`：定义工具、执行工具、记录审计日志，并处理需要确认的动作。
- `llm.py`：封装 OpenAI 兼容聊天接口，实现模型对话、工具调用循环、主动提醒和记忆提取。
- `runtime.py`、`service.py`：装配依赖并协调聊天、设置、工具、记忆和角色状态。
- `qt_app.py`、`ui/`：构建桌面窗口、WebChannel 通信和前端渲染界面。

## 运行链路

用户在桌面窗口输入内容后，前端通过 Qt WebChannel 调用 Python 控制器。服务层刷新桌面上下文和长期记忆，组装系统提示、人设、历史消息和用户输入，再交给大模型接口。模型如果需要执行工具，会返回工具调用请求；工具执行结果以结构化数据回传给模型，模型再生成最终回答。对话结束后，服务层保存对话记录，并按设置提取长期记忆候选。

## 智能体能力

- 感知：桌面上下文被压缩为文本，作为模型理解当前任务的补充信息。
- 记忆：稳定偏好和项目背景写入本地 JSON 文件，后续对话自动注入。
- 行动：工具以明确 schema 暴露给模型，执行结果统一写入审计日志。
- 安全：打开路径、打开链接、启动应用、记忆写入等动作可以按权限配置为允许、询问或拒绝。
- 编排：服务层负责把模型、工具、记忆、上下文和前端状态连接成完整工作流。

## 工具安全设计

项目默认只提供低风险工具。对可能影响本机环境的动作，工具层会根据权限设置返回确认请求，由界面显示给用户。确认通过后，服务层只执行一次原始工具请求。所有工具结果都会写入 `logs/tool_calls.jsonl`，便于排查和复盘。

敏感信息保护主要体现在三个方面：`.env` 不进入仓库；长期记忆会拒绝明显密钥、密码、验证码等内容；默认不把截图图片发送给模型，只传结构化文本上下文。

## 可扩展方向

- 增加新的只读上下文来源，例如剪贴板、浏览器标签页或当前项目文件摘要。
- 增加新的工具时，先定义 schema，再实现处理函数，并为权限策略补充测试。
- 接入新的大模型供应商时，保持与现有流式聊天接口一致，减少服务层改动。
- 增加新的角色模型格式时，在模型扫描模块中添加解析器，并通过模型注册表暴露给界面。
